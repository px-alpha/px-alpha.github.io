The making of this site

I thought it help me to remember what I’ve used for this site, in case I want to change things on it later. So here’s a list and description of what was used for it.1

Jekyll

The site is a Jekyll site. Jekyll is a framework to create static sites, so the only thing that is on the web-server are static HTML pages. Jekyll is a Ruby application that transforms the posts written in Markdown into the aforementioned HTML pages.
Read more: https://jekyllrb.com/, and on extending the Jekyll system without plugins: https://jekyllcodex.org/.

Markdown

The language in which the Jekyll pages are written.
Find a lot more here: https://daringfireball.net/projects/markdown/syntax. And find (warning: geek-mode: on) markdown mode for Emacs: https://jblevins.org/projects/markdown-mode/.

Google fonts

I used Volkorn 2 for the initial character of the main post text.
Find it here: https://fonts.google.com/specimen/Vollkorn

What I used to make this work was

Some CSS and some more CSS, because the drop-caps are rendered differently in Chrome and in Firefox.
Some Kramdown IALs

Page comments

Boy, that turned out to be much more complex than I expected it to be. I mean, how hard can this be in 2020? I tried several solutions, and finally settled on Commentics.

Commenting solutions that did not work out

I tried several solutions that, for various reasons, did not work out. I found several solutions here.

StaticMan

Find it here: https://staticman.net/ and more here https://www.vojtechruzicka.com/staticman/.3 StaticMan keeps your Jekyll site static, by pushing user comments to the GitHub site where your Jekyll code lives and updates the posts accordingly to the comments, by entering pull requests for them (I think, I never got to that point). What stopped me was the unclarity of the documentation: run this on Heroku? Or on GitHub? Which StaticManBot to use? And in which usage scenario? Anyway, I did not make it to get it into a working state.

HashOver

Find it here: https://www.barkdull.org/software/hashover. APHP solution to comments (my provider allows running PHP scripts). Got very confused about names of files and the contents of them (hashover.php in root folder is called /hashover/comment.php in documentation? There are differences between the .zip download and the GitHub repo?). Had to walk away, sorry for that.

Disqus

Find it here: https://disqus.com/.4 A hosted solution. Got it working quite quickly, however, as the solution is based on an iframe that is loaded from the disqus site, there is no way to style the comment area. And it looked horrible. A no-go.

The one that worked: Commentics

Find it here: https://www.commentics.org/. PHP based, easy install, worked right out if the box, even using the iframe setup, the comments area can be styled. Nice. Works well together with Jekyll. There is was, after one full day of work, a working comment solution.

What I learned along the way

Using PHP in a Jekyll site.

An interesting approach, that allows PHP inside Jekyll markup, but also Jekyll markup in PHP, read the article here: Running a combined Jekyll/PHP site on XAMPP by Christian Specht.

Heroku

I knew about Heroku as a Ruby hosting solution, but as a result of the StaticMan research I had a better look at it again. It’s not only Ruby, but all of the other main languages as well. Including CI/CD pipelines et cetera. This may be an interesting solution for future development work.
